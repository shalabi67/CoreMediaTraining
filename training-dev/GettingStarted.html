Core.Dining WAD/AWAD Training - Getting Started

Core.Dining is the CoreMedia training application. The following sections describe how the Core.Dining application is build and started. Please read the instructions carefully.

2. Building the workspace

In the directory workspace execute the following maven command:

          mvn clean install -DskipTests

3. Setup the databases

You need a PostgreSQL instance running on localhost. The following schemes are required (passwords are the same as the scheme name, e.g 'cms' for scheme 'cms').

Database configuration

Encoding:UTF-8
JDBC URL:jdbc:postgresql://localhost:5432/coredining
Schemes:
Content Management Server
Scheme name:cms
Owner(Password):cms(cms)
Master Live Server
Scheme name:mls

Replication Live Server
Scheme name:rls
Owner(Password):rls(rls)

4. Starting the components

Quick start

For starting the components use the following scripts. All server components are started in there own command window using the appropriate maven command. To stop a component simply close the corresponding command window.

00. build workspace.bat
This script executes mvn clean install -DskipTests within the workspace/ directory.
01. start servers.bat
Starts the server components. In the default configuration only the Solr and the Content Management Server web application are started. But you can edit the start script to start the Master-Live and the Replication Server as well.
02. import test-data.bat
Imports training content, users, groups and rights into the Content Management Server. The import data is located at workspace/test-data.
03. start cae webapps.bat
Starts the CAE Preview Web Application. This script is also prepared for starting the live CAE. Just uncomment the appropriate lines.
04. start site-manager.bat
Starts the Site Manager Component (a.k.a CoreMedia Editor)
05. approve and publish.bat
A script used to approve and publish all core.dining content.

Starting single components

There is also a run.bat script in every web application module for starting the corresponding web application.

workspace/modules/server/content-management-server-webapp/run.bat
Starts the Content Management Server.
workspace/modules/server/master-live-server-webapp/run.bat
Starts the Master Live Server.
workspace/modules/server/replication-live-webapp/run.bat
Starts the Replication Live Server.
workspace/modules/search/solr-webapp/run.bat
Starts the Solr Web Application.
workspace/modules/cae/cae-preview-webapp/run.bat
Starts the CAE Preview Web Application.
workspace/modules/cae/cae-live-webapp/run.bat
Starts the CAE Live Web Application.
workspace/modules/cae/studio-webapp/run.bat
Starts the Studio Web Application.
